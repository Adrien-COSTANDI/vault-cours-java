### Rappel des évènements précédents :

Dans la section sur [[14. Les objets (Partie 1)|les objets (partie 1)]] nous avons vu comment créer une classe. On sait que :
- ça sert à **représenter des données** plus complexes que de simples types primitifs grâce à ses **champs/attributs**
- ça sert à **définir des comportements** associés à la manipulation des données grâce à ses **méthodes**
- une **instance** d'une classe a des valeurs assignées grâce à un **constructeur**
- la méthode `{java}public String toString()` permet permet de définir le comportement à avoir lorsqu'on veut afficher une instance de notre classe, et son comportement par défaut est moche

Maintenant, on va rentrer dans des notions liées aux classes, mais qui sont des subtilités dans l'usage qu'on en fait. Ces notions restent tout de même très importantes, puisque vous les retrouverez à peu près partout.
# static

Dans une classe, on a vu qu'on pouvait définir des attributs et des méthodes pour les instances. Mais comment on fait si on n'a pas besoin d'une instance dans certains cas ? Ou alors que certaines données ne concernent aucune instance en particulier ?

Tenez, par exemple, si on prend une classe pour représenter une commande dans un magasin de e-commerce :

```java title:Order.java
public class Order {

  private static long nextId = 1; // partagé par toutes les instances
  private long id;
  private String customerName;

  public Order(String customerName) {
    this.id = Order.nextId; // accès à la variable statique
    Order.nextId++; // on passe au suivant pour le prochain
    this.customerName = customerName;
  }

  public static void skipNextId() {
    Order.nextId++; // celui-là ne sera pas utilisé
  }

  public long getId() {
    return id;
  }

  public String getCustomerName() {
    return customerName;
  }

  @Override
  public String toString() {
    return "Order #" + id + " for " + customerName;
  }
}
```

Pour accéder aux champs `static` d'une classe, on passe par le nom de la classe puis le nom de l'attribut ou de la méthode à utiliser.



---
- [ ] static (un peu final pour pas réassigner)
- [ ] visibilité (private/public/package)
- [ ] héritage (protected + Object) (sealed, @Override)
- [ ] final (pas d'héritage, pas d'override)
